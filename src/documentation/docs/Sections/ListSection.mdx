## Properties

<PropsTable>
  <PropsTableRow value="items" type="Item[]" required>
  Items that should be shown in the list (title: string, description: string, price: number, image: Image, url: string), e.g. ProductListItem.
  </PropsTableRow>
  <PropsTableRow value="headline" type="string" required>
  Specify a headline for the item list. The separator will not be displayed, if no headline is specified.
  </PropsTableRow>
  <PropsTableRow value="filters" type="<string,string>[]" optional>
  A list of filters (key: string, value: string).
  </PropsTableRow>
  <PropsTableRow value="selectedFilters" type="string[]" optional>
  Determine which filters should be selected from the beginning.
  </PropsTableRow>
  <PropsTableRow value="handleFilterChange" type="function" optional>
  A callback that will be invoked when the selected filters change. The new selected filters will be passed as an argument.
  </PropsTableRow>
  <PropsTableRow value="handleItemClick" type="function" optional>
  A callback that will be invoked when a list item is clicked. The clicked item will be passed as an argument.
  </PropsTableRow>
</PropsTable>

## Slots

<SlotsTable>
  <SlotsTableRow value="item" prop="Item" required>
    This slot describes how your items will be rendered.
  </SlotsTableRow>
  <SlotsTableRow value="headline" prop="String">
    This slot optionally replaces the default rendering of the headline.
  </SlotsTableRow>
  <SlotsTableRow value="filter" prop="Object" optional>
    You can also define how the filter buttons get rendered. 
    When the component uses your slot it passes in the following props.
    <span class="code">key</span> the key as defined in your filters.
    <span class="code">value</span> the value as defined in your filters.
    <span class="code">handleClick</span> a callback that you should call when your button gets clicked. It expects the key.
    <span class="code">selected</span> a boolean that describes whether this key is selected or not.
  </SlotsTableRow>
</SlotsTable>
